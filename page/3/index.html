变量名的力量

发表于 2019-11-21 更新于 2020-04-18 分类于 编码技巧

1.变量名的力量

  • 为变量命名时，该名字要完全，准确地描述出该变量所代表的食物

  • 以问题为导向

  • 最适当的平均长度

    • 平均10~16个字符
    • 平均8~20个字符

  • 变量名对作用域的影响

    • 较长的名字适用于很少用到的变量或者全局变量
    • 较短的名字适用于局部变量或者循环变量

  • 变量名中的计算值界定

    • 类似于：Total，Sum，Average，Max，Min，Record，String，Pointer。把界定词加到名字的最后
    • Num限定词例外

  • 变量名中的常用对仗词

    • begin/end
    • first/last
    • locked/unlocked
    • min/max
    • next/previous
    • old/new
    • opened/closed
    • visible/invisible
    • source/target
    • source/destination
    • up/down

2.为特定类型的数据命名

1.循环下标命名

  • 简单的循环，且循环外不使用。用i,j,k即可
  • 嵌套循环和循环外要使用的。使用一个更具有描述性的名字

2.状态变量命名

  • 取一个比flag更具有描述性的名字

3.为临时变量命名

  • 警惕”临时”变量
  • 不要随意使用temp之类的变量名，应该给他取个更好的名字

4.为布尔变量命名

1，谨记典型的布尔变量名

- done:事情完成前设为false,完成后设为true
- error:错误发生前设为false,发生后设为true
- found:找到值设为false,找到设为true
  • sucess或ok:操作失败设为false,成功设为true
    - 如果完成处理表示这个程序执行成功，可以用更具有代表性的名字像：processingComplete
- 如果找到一个值代表成功，可以使用found来替换

2，给布尔变量赋予隐含”真/假”含义的名字

3，使用肯定的布尔变量名

  • 否定的名字如notFound，notDone等，比较南阅读，尤其是取反的时候：if not notFound，应该替换为found

5.为枚举变量命名

  • 在使用枚举类型的时候，可以通过使用组前缀，如：Color_等
  • 具体语言，具体分析，例：Color.Color_Red就没有Color.Red好

6,.为常量命名

  • 根据常量表示的含义进行命名

Maven依赖范围

定义:

用来控制依赖与这三种classpath(编译classpath,测试classpath,运行classpath)的关系。

  • comlile:

若没有指定，则对编译，运行，测试都有效。

  • test:

主代码无法使用此类的依赖，在编译测试代码和运行测试代码的时候使用。

  • provided:

测试和测试classpath有效，但运行的时候无效。

  • runtime:

对于测试和运行classpath有效，编译主代码时无效。

  • system:

依赖关系与provided一致，但必须通过systemPath显示的指定依赖文件的路径

关系表格:

依赖范围(Scope) 对于编译 对于测试 对于运行 例子
complie Y Y Y spring-core
test- — Y — JUnit
provided Y Y — servlet-api
runtime — Y Y JDBC驱动实现
system Y Y — 本地的，Maven仓库之外的类库文件

Maven传递性依赖

--- ### CentOS7安装MySQL(以5.7版本举例，其他版本以此类推)

1.去下载MySQL5.7安装包，直接点击No thanks,just start my download就可以开始下载了。

2.通过rpm -qa | grep mariadb命令查看 mariadb 的安装包

3.通过rpm -e mariadb-libs-5.5.56-2.el7.x86_64 --nodeps命令装卸 mariadb。

4.通过 cd /usr/local/ 命令进入根目录下的usr目录下的local目录，并创建mysql文件夹mkdir mysql

5.把下载好的安装包上传到mysql文件夹,通过tar -xvf mysql-5.7.27-1.el7.x86_64.rpm-bundle.tar解压安装包。

6.解压好后，依次执行如下命令。

rpm -ivh mysql-community-common-5.7.27-1.el7.x86_64.rpm --nodeps --force

rpm -ivh mysql-community-libs-5.7.27-1.el7.x86_64.rpm --nodeps --force

rpm -ivh mysql-community-client-5.7.27-1.el7.x86_64.rpm --nodeps --force

rpm -ivh mysql-community-server-5.7.27-1.el7.x86_64.rpm --nodeps --force

7.依次执行如下命令，执行MySQL的初始化和相关配置。

mysqld --initialize;

chown mysql:mysql /var/lib/mysql -R;

systemctl start mysqld.service;

systemctl enable mysqld;

8.通过 cat /var/log/mysqld.log | grep password命令查看MySQL数据库的密码

9.之后同Windows一样登录MySQL即可,修改密码指令也和Windows一样。
