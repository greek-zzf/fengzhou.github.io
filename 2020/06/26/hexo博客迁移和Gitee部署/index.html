hexo博客迁移和Gitee部署

发表于 2020-06-26 更新于 2020-06-30 分类于 小技巧

博客迁移

最近因为一直想要重装电脑系(爱折腾)，但是又担心博客丢失，不知道如何下手。所以一直没有重装，重装电脑之前看了网上的一些博客迁移教程，跟着做，并进行了对比，结合自己的情况，终于完成了博客的迁移。该方法适用于：电脑重装，或者想在另一台电脑上继续管理博客等。

1.迁移之前的状态

之前使用Hexo博客系统成功部署到GitHub上，可以正常通过 {用户名}.github.io访问博客。

2.迁移之前的准备

我们只需要复制以下几个原来Hexo配置好的博客目录

3.开始迁移博客

  • 在新的系统或者新电脑上，下载并安装Node.js，并配置好环境。使用代码：npm install -g hexo安装hexo，并手动配置好Hexo环境。

  • 在任意盘符建一个新文件夹，如：blog。把之前复制的目录全部放进blog文件夹

  • 进入blog目录，使用命令安装模块npm install，等待安装成功即可。在使用命令安装支持Git的部署插件npm install hexo-deployer-git --save，同样等待安装成功即可。

  • 补充说明;由于是新的环境还需要安装Git工具和配置好Git的全局用户名和邮箱。配置以后博客就可以恢复了。

4.测试发布，部署

通过命令hexo generate编译，再使用命令hexo deploy部署到GitHub。完成。

Hexo博客部署到Gitee(码云)

之前一直都是部署到GitHub,由于国内访问比较慢，所以打算在GItee上面也部署一个，方便自己和他人访问。

1.准备工作

  • 注册一个Gitee账号

  • 新建一个仓库，仓库名称就是你的用户名称。这样地址栏可以直接输入 {用户名}.gitee.io访问。

  • 配置SSH公钥(具体的使用可以查百度，很简单的)

2.部署步骤

github字段后面是github仓库对应的ssh地址，gitee字段后面对应的是gitee的仓ssh库地址。默认分支都是master。配置好之后，直接编译部署。

3.使用Gitee Pages

在Gitee上查看是否部署成功，要看到效果需要开启Gitee Pages服务。点进仓库，点击服务，点击Gitee Pages，然后选择要部署的分支，由于我们是使用的master所以选用master就可以了。底部有强制使用HTTPS勾选框，打钩(之前没有打钩遇到主题样式丢失问题和页面失效的问题，打钩之后就没有遇见),然后点击更新等待即可。成功之后就可以访问了。

注：Gitee Pages每次博客更新都需要自己手动更新一次，即重复一次第三步的操作。GitHub会自动更新，当然也可以付费使用Gitee Pages Pro，即可自动更新。
